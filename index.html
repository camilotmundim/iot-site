Problema Investigado

O Tribunal de Justiça do Estado de Rondônia possui um processo manual para associação de guias avulsas de custas processuais aos respectivos processos judiciais.

Embora o sistema de custas disponibilize funcionalidades para realizar essa associação, o fluxo operacional ainda depende integralmente da intervenção humana para:

  • Identificar o processo judicial correspondente;
  • Localizar o boleto associado;
  • Extrair o “Nosso Número”;
  • Realizar a vinculação manual;
  • Validar inconsistências entre sistemas.
O problema central identificado foi a existência de um microprocesso repetitivo, operacional e altamente dependente de intervenção humana, gerando desperdício de tempo, baixa escalabilidade e risco de inconsistência de dados.

Produto Técnico Proposto

O produto desenvolvido consiste em uma plataforma inteligente de automação orientada a eventos para processamento e associação automática de guias avulsas.

Mensageria

O fluxo inicia automaticamente a partir do recebimento de eventos em filas de processamento assíncrono.

Integração entre Sistemas

A solução realiza comunicação automatizada entre múltiplas APIs e sistemas do ecossistema judicial.

Inteligência Artificial

Um modelo de Machine Learning identifica automaticamente documentos relacionados a custas processuais.

Extração Inteligente

O sistema extrai automaticamente o “Nosso Número” diretamente do conteúdo textual do documento.

Automação do Processo

A associação da guia ao processo judicial ocorre automaticamente sem intervenção humana.

Escalabilidade

A arquitetura distribuída permite crescimento contínuo do processamento sem aumento proporcional de esforço humano.

Arquitetura Tecnológica

1. Recebimento do evento via fila de mensageria

2. Consulta à API de documentos processuais

3. Extração textual automatizada do documento

4. Classificação documental via modelo de Machine Learning

5. Identificação do “Nosso Número”

6. Consulta ao sistema de custas

7. Validação da guia do tipo “Avulsa”

8. Associação automática da guia ao processo judicial

Fluxo Automatizado

Evento Recebido

Uma fila informa a existência de um novo documento processual para análise.

Extração do Documento

O sistema consulta uma API para obter o conteúdo textual do documento judicial.

Classificação Inteligente

O modelo de Machine Learning verifica se o documento pertence à categoria de custas processuais.

Identificação do Nosso Número

Um algoritmo realiza a extração automática do identificador financeiro do boleto.

Validação da Guia

O sistema verifica automaticamente se a guia pertence ao tipo “Avulsa”.

Associação Automática

O vínculo entre a guia e o processo judicial é realizado automaticamente via integração sistêmica.

Interoperabilidade

Impactos Esperados

  • Redução significativa do tempo gasto na associação manual;
  • Diminuição de inconsistências entre sistemas;
  • Redução do retrabalho operacional;
  • Aumento da integridade dos dados financeiros;
  • Maior escalabilidade do fluxo de custas processuais;
  • Liberação de servidores para atividades de maior valor agregado;
  • Fortalecimento das iniciativas de Justiça Digital e Justiça 4.0.
